# 机房动力设备智能盘点系统 - 实施计划

## 0）项目全貌（先看这个）

本项目是一个“机房动力设备盘点”MVP系统，目标是让一线盘点人员尽量只做“拍照/上传”动作，系统自动调用视觉大模型识别设备铭牌信息（如品牌、型号、生产日期等），并沉淀为可汇总、可追溯、可人工确认版本的真实台账。

### 项目目标
- 解决人工填报不准确、难核验的问题
- 以“站点”为母关系沉淀设备资产信息
- 支撑“超期服役”分析与更新计划的数据基础

### 技术范围（当前约定）
- 前端：Vue（响应式，兼容手机与PC上传）
- 后端：FastAPI
- 识别能力：调用现有外部视觉模型API（由业务方提供）
- 权限：MVP阶段不做权限体系
- 登录策略：首次免密，第二次起需密码；未配置密码时持续提示配置

### 关键业务规则（冻结）
- 重复任务判定：地市 + 站点 + 时间（天） + 盘点设备类型 相同
- 盘点更新判定：地市 + 站点 + 设备类型 相同，时间（天）不同
- 汇总使用规则：系统可建议“最新时间”版本，但必须人工确认后才生效
- 首页待办：必须有“待确认更新任务”入口，供人工判断采用哪个版本
- 判重时间标准：YYYY-MM-DD
- 有效版本状态位：`pending_confirm` / `active` / `inactive`
- 首次登录免密码；从第二次登录开始需要密码
- 若未配置登录密码，系统每次进入主页均提示去“配置页面”设置密码，直到配置完成

### 数据录入策略（动态沉淀）
- 无完整站点主数据：采用“录入即入库，下次下拉复用”
- 该策略可复用于其他基础字段（如地市、设备类型等）

### 交付形态（MVP）
- 可创建任务、可上传（拍照/多文件）
- 可调用识别并保存结构化结果
- 可识别更新任务并进入首页待办
- 可人工确认当前有效版本
- 可生成并导出汇总结果

> 说明：本计划用于跟踪开发进度。状态分为：`[x] 已完成`、`[ ] 未完成`。
